# Security Policy

## Supported Versions
Security fixes are provided for the latest released version only.

## Reporting a Vulnerability
Please do **not** report security vulnerabilities via public GitHub issues.

Use one of the following private channels:
1. **GitHub Security Advisories** (preferred), if enabled for this repository.
2. **Private Vulnerability Reporting** on GitHub (if available/enabled).

If neither option is available, open a GitHub issue **without sensitive details** and request a private channel from the maintainers.

## What to include
- A clear description of the vulnerability and impact
- Steps to reproduce (proof of concept if possible)
- Affected version(s) and environment details (OS, app version, Electron/Node version)
- Any suggested mitigation

## Response Timeline
We aim to acknowledge valid reports within **72 hours** and provide a remediation plan based on severity and complexity.

## Coordinated Disclosure
We follow coordinated vulnerability disclosure practices. Please allow a reasonable time to address the issue before public disclosure.
